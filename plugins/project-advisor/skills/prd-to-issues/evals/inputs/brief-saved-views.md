# Feature Brief: Gespeicherte Ansichten fuer Personenlisten

## Kontext

Unser internes Operations-Team arbeitet taeglich mit einer Personenliste, auf der viele Filter gleichzeitig gesetzt werden. Heute muessen Nutzer dieselben Filter nach jedem Seitenwechsel oder nach einer erneuten Anmeldung wieder von Hand setzen.

## Ziel

Nutzer sollen haeufig genutzte Filterkombinationen als persoenliche Ansicht speichern und spaeter mit wenigen Klicks wiederverwenden koennen.

## Gewuenschtes Verhalten

- Ich kann meine aktuelle Filterkombination auf der bestehenden Personenliste speichern.
- Ich vergebe dabei einen Namen fuer die Ansicht.
- Ich kann spaeter eine meiner gespeicherten Ansichten erneut laden.
- Wenn eine gespeicherte Ansicht Filter enthaelt, die nicht mehr gueltig sind, soll das nicht still scheitern.

## Randbedingungen

- Die erste Version ist nur fuer angemeldete interne Nutzer gedacht.
- Gespeicherte Ansichten sind persoenlich und werden nicht mit anderen Nutzern geteilt.
- Das bestehende Listen- und Filtererlebnis soll erweitert, nicht ersetzt werden.

## Non-Goals

- Kein Teilen von Ansichten zwischen Nutzern.
- Keine globale Admin-Verwaltung fuer gespeicherte Ansichten.
- Keine komplett neue Listenoberflaeche.
